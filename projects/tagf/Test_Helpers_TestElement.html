#coerce_to(coercer, val) ⇒ Any Originally defined in module RoUS::TestHelpers

Coerce a String into another class. Helper designed to allow us to display a value in a canonical form rather than the one to which Ruby might reduce it. For example, '00' ~= 00.to_s.

Parameters:

  • coercer (Class, Method, Proc) —

    Class to which the value should be cast, or a method/proc that will do the coercion.

  • val (String) —

    The string representation of the item to be coerced.

Returns:

  • (Any) —

    The value returned by the coercing method or class, or the unmodified input value if coercer is nil.

Raises:

  • (ArgumentError) —

    coerce_to requires an object of type Class, Proc, or Method

  • (ArgumentError) —

    only String objects are supported for coercion: val.class:val.inspect

  • (ArgumentError) —

    objectcoercer-class:coercer.inspect does not support casting/coercion`

#mktestname(**kwargs) ⇒ Symbol Originally defined in module RoUS::TestHelpers

Helper method to generate a dynamic test name (symbol). Since Unit::Test invokes test methods using __send__, names are not limited to [_[:alnum:]]* and we don't have to pussyfoot around things like floating-point or complex number representations.

   testval: '(0.0-1.1i)', coerce: Complex

Parameters:

  • kwargs (Hash<Symbol=>Object>) —

    All arguments are passed by position-independent keyword.

Options Hash (**kwargs):

  • :testval (Object) —

    The value under test, which will be built into the test name.

  • :coerce (Class) —

    Sometimes the :testval is passed as a string representation to retain syntax, but needs to be coerced back to the original class in order to derive its class. As an example, (0.0-1.1i).to_s evaluates to "0.0-1.1i". If we want the test name to include the original, we would use

  • :expected (Boolean) — default: true —

    The result expectation for the test being named.

  • :prefix (String) — default: "for" —

    Any additional explanatory text between the prefix and the test value.

  • :suffix (String) — default: kwargs[:testval].inspect —

    Text to use as the suffix of the test name, overriding the calculation of a derived value ("testval-class_testval.inspect").

Returns:

  • (Symbol) —

    A symbol suitable for use as a Test::Unit test method name, built from "test_prefix_expected_suffix"

def test_013_default_render_string_tracks
  te			= TestElement.new(value: 'Lorem ipsum')
  assert_equal('String_"Lorem ipsum"',
               te.render,
               'te.render correct default (String)'
              )
  te.send(:value_set=, false)
  newval		= 'E Power Biggs'
  te.value		= newval
  assert_equal(format('String_%s', newval.inspect),
               te.render,
               'te.render value change tracked (String)'
              )
end

def test_050_klass_attribute
  te1			= TestElement.new(value:	1,
                                        :attrscope => :class)
  te1.test_attr	= 'te1.test_attr'
  assert_equal('te1.test_attr',
               te1.test_attr,
               'accessing dynamic klass variable from 1st instance')
  te2			= TestElement.new(value: 1)
  assert_equal('te1.test_attr',
               te2.test_attr,
               'accessing dynamic klass variable from 2nd instance')
  assert_equal('te1.test_attr',
               TestElement.test_attr,
               'accessing dynamic klass variable from class')
  assert_true(te1.klass_attributes.include?(:test_attr),
              'Verify addition of attribute to klass_attributes')
end
